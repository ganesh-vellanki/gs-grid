Get started

Set up gs-grid, configure columns and data, and wire the component into your page with the runtime APIs that are available today.

Prerequisites

Before mounting the grid, make sure your environment can compile the package and run Web Components.

  • Node.js 14+ and Yarn 1.x
  • A browser with Web Components support
  • Basic familiarity with custom elements and standard DOM access
Run locally

The local demo site builds the library, generates docs, and serves the example app so you can validate configuration changes quickly.

1. Install

Pull all dependencies required for the grid, docs, and example app.

2. Build

Compile the library and generate the latest documentation output.

3. Serve

Start the demo so you can verify the configuration and runtime APIs visually.

yarn install
yarn start
1. Install in your app

Install the package first, then import the component and model helpers in the part of your app where the grid is initialized.

yarn add @gs-grid/gs-grid
2. Render the grid element

Render the custom element wherever you want the grid to appear. The component receives its runtime configuration after it mounts and exposes its generated instance-id.

<gs-grid></gs-grid>
3. Configure the grid step by step

Instead of one large code block, the setup below is split into smaller pieces so each part of the configuration is easier to understand.

Step A: Import the component and helpers

Import the custom element once so the browser can register <gs-grid>. Then import the model helpers used to build and dispatch the runtime configuration.

import '@gs-grid/gs-grid';
import { GridConfig, GridEvents } from '@gs-grid/gs-grid/dist/model';
Step B: Create the base config

Start with a new GridConfig. This is where you define row height, optional row-selection support, and the rest of the runtime behavior for the grid instance.

const gridConfig = new GridConfig();
gridConfig.rowHeight = 32;
gridConfig.enableRowSelection = true;
Step C: Define columns

Each column controls which field is rendered, what text appears in the header, whether sorting is enabled, and how width should be allocated. Nested values can be read with dot notation such as company.name.

gridConfig.columnDefs = [
    {
        field: 'name',
        headerName: 'Name',
        enableSort: true,
        width: 180
    },
    {
        field: 'email',
        headerName: 'Email',
        enableSort: true,
        width: '30%'
    },
    {
        field: 'company.name',
        headerName: 'Company',
        enableSort: true,
        width: 180
    },
    {
        field: 'status',
        headerName: 'Status',
        width: 120
    }
];
Step D: Provide row data

Assign any array of plain objects. The field names from columnDefs are resolved against each row, including nested objects.

gridConfig.data = [
    {
        name: 'Ava Patel',
        email: 'ava@example.com',
        company: { name: 'Northwind' },
        status: 'Active'
    },
    {
        name: 'Lucas Reed',
        email: 'lucas@example.com',
        company: { name: 'Contoso' },
        status: 'Pending'
    }
];
Step E: Listen for selection changes

If row selection is enabled, you can register an async callback to receive the full selected-row list and the row that triggered the latest change.

gridConfig.onRowSelectionChangeAsync = async (selectedRows, currentRow) => {
    console.log('selected rows', selectedRows);
    console.log('current row', currentRow);
};
Step F: Attach the config to the rendered element

After the element is in the DOM, read its generated instance-id and dispatch the config through GridEvents.setupGridConfig(...). That is what activates the grid.

const gridEl = document.querySelector('gs-grid');
const instanceId = gridEl?.getAttribute('instance-id');

if (gridEl && instanceId) {
    GridEvents.setupGridConfig(instanceId, gridConfig);
}
4. Use runtime APIs

Once the grid is configured, you can trigger common operations without reconstructing the component. These APIs are useful for search boxes, toolbar buttons, export actions, and row-selection controls.

Search

Search across all configured fields, then clear the active query when needed.

await gridConfig.performSearch?.('northwind');
await gridConfig.clearSearch?.();
Export

Export the current grid dataset to CSV or Excel directly from the bound runtime helpers.

await gridConfig.exportToCsv?.('customers');
await gridConfig.exportToExcel?.('customers');
Read selection

Access the current selection at any time through the grid instance API.

const selectedRows = gridConfig.instance.getSelectedRows();
Toggle selection

Enable or disable row-selection controls at runtime without rebuilding the grid.

await gridConfig.instance.setRowSelectionEnabled(false);
