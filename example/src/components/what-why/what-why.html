Why

gs-grid is a lightweight grid built as a Web Component, so it can run in plain JavaScript, Knockout, React, Angular, Vue, or any app that can render custom elements.

The goal is simple: fast rendering, easy setup, and framework-agnostic integration.

What It Gives You
  • Custom-element grid tag: <gs-grid>
  • Column definitions with auto, fixed, percent, and flexible width options
  • Configurable row height and nested field access (e.g. company.name)
  • Built-in column sorting with ascending, descending, and reset states
  • Optional row selection with header select-all, indeterminate state, and selected-row access
  • Shadow DOM style isolation for predictable rendering
Why This Approach
  • No framework lock-in for teams with mixed stacks
  • Event-driven setup keeps usage straightforward
  • Virtualized row rendering keeps large datasets responsive
  • Custom smart scroll behavior supports drag, click, and wheel
  • Convenience APIs make search, export, and selection easy to trigger at runtime
Available Features
  • Framework-agnostic Web Component usage
  • Column definitions with nested-field support
  • Per-column sorting with visual header indicators
  • Global search across all configured fields
  • CSV export from the current grid dataset
  • Excel export from the current grid dataset
  • Virtualized rendering with viewport-based row windows
  • Custom smart scrollbar with drag, click, and wheel behavior
  • Responsive viewport height recalculation on resize
  • Optional row selection with row checkboxes and header checkbox
  • Async row-selection change callback support
  • Runtime APIs to search, export, read selected rows, update data, and enable or disable row selection
How It Works
  1. Render <gs-grid> in your page.
  2. Create a GridConfig with columnDefs, rowHeight, data, and optional features like enableRowSelection.
  3. Dispatch configuration using GridEvents.setupGridConfig(instanceId, config).
  4. The component renders header, viewport rows, selection UI, and custom scrollbar inside Shadow DOM.
  5. Scroll position is mapped to virtual row windows for efficient updates, while search, sort, export, and selection APIs remain available at runtime.
Best Fit

gs-grid is currently a strong fit for:

  • Internal tools and dashboards
  • Data-heavy pages that need virtualization without a large grid dependency
  • Apps that need a custom-grid experience without heavy dependencies
  • Teams that want a framework-agnostic grid with runtime APIs for search, export, and selection
